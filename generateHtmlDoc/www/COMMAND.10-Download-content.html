Overview

This command downloads the content. The download command will work ONLY if it refers a valid authorization.

URI

/MMS_0000/CatraSoft/000/000/561/833_source.mp4?:token&:outputFileName

Method

GET

URL Params

Required

  • token: it is generated by the create delivery authorization API
  • example: token=654
  • outputFileName: it is metadata field decided by the user during the ingestion of the content. It is used by the platform to set the HTTP 'Content-Disposition' to give a name to the downloaded file name.
  • example: outputFileName=myName.mp4

Optional

None

Payload Data

None

Success Response

Code

200 OK

Content

Media file

Error Response

See Common Error Responses

Sample Call

Assuming: * Token: 765 * OutputFileName: myName.mp4

curl "https://mms-delivery.catrasoft.cloud/MMS_0000/CatraSoft/000/000/561/833_source.mp4?token=6&outputFileName=myName.mp4" > myName.mp4

In case the URL (https://mms-delivery.catrasoft.cloud/MMS_0000/CatraSoft/000/000/561/833_source.mp4?token=6&outputFileName=myName.mp4) is set in a browser, the downloaded file will be named by the browser as myName.mp4.

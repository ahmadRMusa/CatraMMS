Registration API.htmls:

  1. register a new User (and his Workspace). When a new User is registered, a Workspace dedicated to the User is created as well. Once the User and Workspace are created, the API.html will send an email to the User providing a confirmation code that has to be used by the next API.html.
    1. Register command details (HTTP POST)
  2. confirm the registered User. This API.html will send the confirmation code to the MMS system so that the User/Workspace will be enabled. It will return the API.html Key associated to the User and Workspace. The API.html Key has to be used to call any MMS API.html.
    1. Confirm command details (HTTP PUT)

The API.html Key has to be 'saved' because it has to be used in any MMS API.html.
